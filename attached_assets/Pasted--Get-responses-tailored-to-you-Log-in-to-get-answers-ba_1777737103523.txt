hey how to add scroll in rust ratatui ?

In ratatui, scrolling isn’t something that just “turns on” automatically—you usually manage it yourself by tracking an offset and rendering only the visible slice of content.

Here’s the basic idea:

1. Track scroll state
You need a variable (often in your app state) to store how far the user has scrolled.

struct App {
    scroll: u16,
}
2. Update scroll on input
When handling key events (like ↑ / ↓), update the scroll value:

use crossterm::event::{KeyCode, KeyEvent};

fn handle_input(app: &mut App, key: KeyEvent) {
    match key.code {
        KeyCode::Down => app.scroll += 1,
        KeyCode::Up => {
            if app.scroll > 0 {
                app.scroll -= 1;
            }
        }
        _ => {}
    }
}
3. Apply scroll to a widget
Many widgets support scrolling. For example, with Paragraph:

use ratatui::widgets::{Paragraph, Block, Borders};

let text = "A very long text...\nMore lines...\nEven more lines...";

let paragraph = Paragraph::new(text)
    .block(Block::default().borders(Borders::ALL))
    .scroll((app.scroll, 0)); // (vertical, horizontal)
4. Clamp the scroll (important)
If your content is shorter than the scroll value, things break visually. So clamp it:

let max_scroll = text.lines().count().saturating_sub(height as usize);
app.scroll = app.scroll.min(max_scroll as u16);
5. Alternative: List scrolling
If you're using List, you typically use ListState instead:

use ratatui::widgets::ListState;

let mut state = ListState::default();
state.select(Some(selected_index));
Then update selected_index instead of a raw scroll value.

Common gotchas
Ratatui does not handle scrolling automatically—you must manage state.

You need to know your viewport height to clamp correctly.

Horizontal scroll is the second value: .scroll((y, x)).

Ahh—got it, you want mouse wheel / touchpad scrolling, not just keyboard 👍

In ratatui, mouse scrolling works—but only if you:

Enable mouse capture

Handle mouse events

Update your scroll state

1. Enable mouse capture
Using crossterm:

use crossterm::execute;
use crossterm::event::EnableMouseCapture;

execute!(std::io::stdout(), EnableMouseCapture)?;
Don’t forget to disable it on exit:

use crossterm::event::DisableMouseCapture;

execute!(std::io::stdout(), DisableMouseCapture)?;
2. Listen for mouse events
In your event loop:

use crossterm::event::{self, Event, MouseEventKind};

if let Event::Mouse(mouse) = event::read()? {
    match mouse.kind {
        MouseEventKind::ScrollDown => {
            app.scroll += 1;
        }
        MouseEventKind::ScrollUp => {
            if app.scroll > 0 {
                app.scroll -= 1;
            }
        }
        _ => {}
    }
}
3. Apply it to your widget
Same as before:

Paragraph::new(text)
    .scroll((app.scroll, 0));
Important notes (this is where people get stuck)
Touchpad scroll = mouse scroll events → works automatically once enabled.

If nothing happens:

Make sure your terminal supports mouse events (most modern ones do).

Run inside a proper terminal (not some limited IDE console).

Some terminals require alternate screen mode:

use crossterm::terminal::EnterAlternateScreen;
execute!(stdout, EnterAlternateScreen)?;
Bonus: smoother scrolling
Instead of += 1, you can scroll faster:

app.scroll += 3; // tweak speed
Common pitfall
If scrolling fires but doesn’t move visually, it's usually because:

you didn’t clamp scroll

or content isn’t taller than the viewport
